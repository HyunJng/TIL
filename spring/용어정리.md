# 스프링 사용한 개발하며 많이 듣는 용어 정리

## DAO(Data Access Object)
DB를 사용해 데이터를 조회하거나 조작하는 기능을 전담으로 만든 **오브젝트**

## 느슨한 관계
관계를 유지하는 데 꼭 필요한 최소한의 방법만 간접적인 형태로 제공하고, 
나머지는 서로 독립적이고 알 필요가 없게 만들어주는 것 

## 응집도/결합도
* 응집도⇡: 하나의 모듈, 클래스가 하나의 책임 또는 관심사에만 집중되어있다.
* 결합도⇣: 책임과 관심사가 다른 오브젝트 또는 모듈과는 느슨하게 연결되어 있다.

## 제어의 역전: IoC(Inversion Of Control)
오브젝트가 자신이 사용할 오브젝트가 무엇이고 언제 생성하는지, 심지어 자신이 어디서 생성되고 사용되는지에 관한 
모든 제어권을 자신이 아닌 다른 대상에게 위임하는 것
* 템플릿 메서드 패턴: 서브클래스는 상위클래스에게 제어권을 위임
* 프레임워크: 애플리케이션이 프레임워크에 제어권을 위임

## 라이브러리/프레임워크
> 제어권을 누가 갖고 있느냐가 가장 큰 차이
* 라이브러리: 애플리케이션 코드가 제어권을 갖고 필요한 기능이 있을 때 라이브러리를 사용
* 프레임워크: 프레임워크가 제어권을 갖고 애플리케이션 코드를 실행(제어의 역전)

## 의존성 주입: DI(Dependency Injection)
* 런타임에 실제 "구현체"를 생성하고 의존관계를 맺은 클라이언트에 주소를 전달하여 연결해주는 작업
* DI가 가능한 핵심:
  * **런타임 시점에 오브젝트간의 관계를 맺는 제 3자가 존재**해야한다
  * 즉, Spring은 IoC 컨테이너가 존재하기에 DI가 가능하다.
* DI가 있기에 오브젝트 간에 "느슨한 관계"를 만들 수 있다
